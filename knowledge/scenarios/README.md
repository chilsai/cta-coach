# CTA Practice Scenarios

Each scenario mirrors the format and complexity of a real CTA Review Board case.

|------|-------|----------------|

## Adding New Scenarios

Follow the structure of `scenario-001-global-manufacturing.md`:

1. **Background** — Company size, industry, current Salesforce state
2. **Business Requirements** — 4-6 numbered requirements spanning multiple domains
3. **Constraints** — Budget, timeline, team, technology
4. **Candidate Instructions** — Time allocation and clarifying question allowance
5. **Suggested Clarifying Questions** — What a strong candidate would ask
6. **Scoring Guide** — Green flags and red flags for coaches
7. **Model Solution Highlights** — For coach reference only

## Scenario Difficulty Levels

| Level | Characteristics |
|-------|---------------|
| **Foundation** | 2-3 domains, clear requirements, obvious constraints |
| **Intermediate** | 4-5 domains, some ambiguity, competing constraints |
| **Board-Ready** | 6+ domains, significant ambiguity, political/budget tension, trick elements |

New scenarios should target **Board-Ready** difficulty.
