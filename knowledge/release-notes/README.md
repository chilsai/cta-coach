# Release Notes

This directory is automatically populated by the knowledge updater agent.

Run `/update-knowledge` in a coaching session, or trigger the GitHub Actions workflow manually, to pull the latest Salesforce release notes.

## File Naming Convention

```
YYYY-MM-DD-release-summary.md       # Per-release summary (Spring/Summer/Winter)
YYYY-MM-DD-architect-docs-changes.md  # Changes to architect.salesforce.com
YYYY-MM-DD-coaching-impact.md       # Changes that affect coaching guidance
```

## Salesforce Release Schedule

| Release | Approximate Date |
|---------|----------------|
| Spring  | February       |
| Summer  | June           |
| Winter  | October        |

Run `/update-knowledge` after each release to keep coaching content current.
